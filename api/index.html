SocialStats API Documentation

SocialStats exposes a social-network-agnostic model for profile, post, and comment/reply metrics. Each social network has different labels and available KPIs, but integrations should consume the normalized fields wherever possible.

Base URLhttps://socialstats.axiombilling.com
ProtocolHTTPS
AuthenticationX-API-Key header
ContractProfile, posts, comments, replies

Shared Concepts

ConceptMeaning
Profile URLTarget account, page, channel, or profile URL.
Post URLCanonical URL for a post, reel, video, photo, article, or equivalent content item.
ReactionsLikes, reactions, hearts, thumbs-up, or the closest network equivalent.
CommentsTop-level/main comments on a content item.
RepliesNested responses under a main comment, when the network exposes them.
Views, Shares, RepostsReturned when visible and supported by the network/content type.

For fields that do not exist on a network or are not visible in the authenticated session, responses may return null, an empty array, or "not available" depending on the field family.

Authentication

Every request must include your assigned API key in the X-API-Key header.

X-API-Key: <your-api-key>
Do not place API keys in frontend JavaScript, public repositories, logs, or screenshots. Treat the key as a backend/server-side secret.

Health Check

Use this endpoint to verify that the API is reachable and browser sessions are available.

GET /health

cURL

curl -H "X-API-Key: <your-api-key>" \
  https://socialstats.axiombilling.com/health

Profile & Post Scrape

Use /scrape to collect profile-level data and recent post-level data from a supported social network.

POST /scrape
Content-Type: application/json
X-API-Key: <your-api-key>

Request Body

{
  "platform": "facebook",
  "url": "https://www.facebook.com/example",
  "num_of_posts": 10
}

Parameters

FieldTypeRequiredDescription
platformstringyesOne of instagram, youtube, linkedin, facebook.
urlstringyesFull HTTPS profile/page/channel URL.
num_of_postsintegernoTarget number of recent post/content items. Default: 10.

cURL

curl -X POST https://socialstats.axiombilling.com/scrape \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your-api-key>" \
  -d '{
    "platform": "facebook",
    "url": "https://www.facebook.com/example",
    "num_of_posts": 10
  }'

Post Comments Scrape

The comments contract collects main comments and reply KPIs from post URLs. It expects post URLs, a maximum number of main comments, and an optional comment-selection filter.

PlatformStatus
FacebookImplemented via POST /facebook/comments.
InstagramPlanned under the same response contract.
YouTubePlanned under the same response contract.
LinkedInPlanned where comments are visible.
 
POST /facebook/comments
Content-Type: application/json
X-API-Key: <your-api-key>

Request Body

{
  "post_urls": [
    "https://www.facebook.com/permalink.php?story_fbid=...",
    "https://www.facebook.com/reel/5485480465010309"
  ],
  "max_comments": 10,
  "selection_filter": "Most relevant"
}

Parameters

FieldTypeRequiredDefaultDescription
post_urlsstring[]yesnonePost/content URLs to process sequentially. Current endpoint accepts Facebook URLs.
max_commentsinteger or "all"no10Number of detailed main comments to return per post. Use "all" to return every loaded main comment.
selection_filterstringnoMost relevantAccepted values: Most relevant, Newest, All comments.

Selection Filters

FilterNormalized MeaningScrolling StrategyBest Use
Most relevantPlatform default/top/relevant comment ordering.Fast path. Stops as soon as max_comments main comments are captured.Fast previews and top comments.
NewestNewest-first comment ordering.Stops as soon as max_comments main comments are captured.Recent comment monitoring.
All commentsBroadest available comment set.Scrolls to the end to estimate full main-comment and reply totals, while returning only max_comments detailed comments unless max_comments is "all".Reconciliation and total comment accounting.

cURL

curl -X POST https://socialstats.axiombilling.com/facebook/comments \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your-api-key>" \
  -d '{
    "post_urls": [
      "https://www.facebook.com/reel/5485480465010309"
    ],
    "max_comments": 10,
    "selection_filter": "All comments"
  }'

Platform Reference

Instagram

Supported profile URL format: https://www.instagram.com/<handle>/. Current /scrape support includes profile-level metrics and recent post likes/comments where visible.

[
  {
    "followers": 42000,
    "profile_name": "National Geographic",
    "profile_image_link": "https://cdn.instagram.com/...",
    "profile_url": "https://www.instagram.com/natgeo/",
    "posts": [
      { "likes": 1200, "comments": 45 }
    ]
  }
]

YouTube

Supported channel URL format: https://www.youtube.com/@<handle>. Current /scrape support is channel-level.

[
  {
    "name": "Example Channel",
    "subscribers": 477000000,
    "views": 1211000000,
    "videos_count": 800,
    "profile_image": "https://yt3.googleusercontent.com/...",
    "url": "https://www.youtube.com/@example",
    "top_videos": []
  }
]

LinkedIn

Supported profile URL format: https://www.linkedin.com/in/<profile-slug>/. Current /scrape support includes profile-level metrics and recent activity interaction strings.

[
  {
    "name": "Example Person",
    "followers": 40000000,
    "connections": 500,
    "avatar": "https://media.licdn.com/...",
    "url": "https://www.linkedin.com/in/example/",
    "activity": [
      { "interaction": "Liked by 12 people" }
    ]
  }
]

Facebook

Supported URL formats include profile/page URLs, profile.php?id=..., permalink posts, and reels. Current /scrape support includes profile/page fields and recent posts with post URLs. Current /facebook/comments implements the comments contract for Facebook post URLs.

  • Post URLs are captured from the post date/permalink anchor.
  • date_unix_ms is included when Facebook exposes a parsable relative or absolute date.
  • If Facebook does not expose views for non-video posts, views may be estimated using the average views-to-reactions ratio from posts in the same run that do expose views.
  • Comment text is not collected in /scrape; use the comments endpoint with returned post_url values.

Operational Notes

  • The API uses authenticated browser sessions. If a platform session expires, the platform must be logged in again on the server.
  • Requests for the same platform are processed one at a time for browser session stability.
  • Most relevant and Newest are optimized for speed.
  • All comments is optimized for reconciliation and can take substantially longer.
  • Social network counts can drift while scraping. main_comments_plus_replies_total is a reconciliation value and may differ slightly from total_comments_displayed.
  • Some networks do not expose every KPI. Missing or unavailable values should not be treated as scraper failure unless the endpoint returns an explicit error.
